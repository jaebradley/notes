# [What are software and hardware interrupts, and how are they processed?](https://unix.stackexchange.com/a/18023/459923)

## Hardware Interrupts
* Hardware interrupts are issued by hardware devices - disk, network cards, keyboards, clocks, etc
* Each device will have its own Interrupt Request Line (IRQ)
* CPU dispatches interrupt request to the appropriate hardware driver based on the IRQ
  * Hardware drivers are usually subroutines within the kernel rather than a separate process
* The CPU executes the driver logic that handles the interrupt
* In multiprocessor systems, an interrupt will usually only interrupt one of the CPUs
* Hardware interrupts trigger relevant kernel code to execute
  * This interrupt-related logic may allow itself to be interrupted by other hardware interrupts
  * In the case of a timer-based interrupt, the kernel schedule may suspend the process that was running and allow another process to execute
  * The scheduler logic enables multitasking

## Software Interrupts
* Typically, software interrupts are requests for I/O
  * They can only be generated by currently running processes
  * They are instructions in a user-space program where [the program wants to pass some data to the hardware](https://stackoverflow.com/a/47331677)
* These I/O requests call the kernel, which then schedules the I/O to occur
* For some devices, the I/O is done immediately, but disk I/O is usually queued and completed at a later time
* Depending on the I/O, the process executing the I/O request may be suspended until the I/O completes
  * The kernel scheduler then picks another process to execute
* Software interrupts only communicate with the kernel
* A special software interrupt is a yield call, which request that the kernel scheduler checks to see if another process can execute
