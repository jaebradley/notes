# [Vibe coding and agentic engineering are getting closer than I’d like](https://news.ycombinator.com/item?id=48037128)
* “I don’t know any good engineers just because they wrote code faster”
* “Good engineers won’t continue to be good when vibe-coding, because the thing that made them good was the learning loop”
  * The referenced learning loop is something to the effect of 1. start with a designed solution 2. implementation may identify edge cases / missing paths 3. Go back to the design / step 1 and try again
* “If the code compiles and works, but it does the wrong thing in some edge case, or has a security vulnerability, or introduces tech debt or dubious architectural decisions, that’s harder to spot but doesn’t reduce the review burden whatsoever. If anything “truthy” code is more mentally taxing to review than just obviously bad code.”
* Coding agents get close to a solution on the first one shot, but then require work to get the last X%
  * Coding agents, unlike human engineers, keep the bug / malformed architecture as they continue whereas human engineers may stop every so often to refactor to a reasonable state before continuing to proceed
